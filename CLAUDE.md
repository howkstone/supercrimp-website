# Standing Instructions

At the start of every session, run `git pull origin master` before doing anything else.

Never run `git push` or deploy anything unless the user explicitly says "Deploy".

When the user says "Deploy", push all committed changes to GitHub with a clear commit message summarising what changed.

If a request is ambiguous about which website or file to edit, ask for clarification before making any changes.

The active website is eatcrimp.uk. All edits should target the eatcrimp-website repo unless explicitly told otherwise.
